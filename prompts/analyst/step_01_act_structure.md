# Analyst — STEP 1: 막 구조 및 분량 설계

## 입력 컨텍스트

다음 파일을 읽어라:
- `projects/{name}/config.yaml` — target_duration 확인
- `projects/{name}/output/step_00_critique.md` — STEP 0 보완점 목록

## 작업 지시

시놉시스를 **3막 구조로 재배치**하고 **분량을 설계**하라.

### 1단계: 3막 구조 설계

Syd Field 3막 구조를 기반으로, target_duration에 맞게 설계하라.

**90분 기준 4등분 배치:**

| 막 | 비율 | 시간 | 씬 수 | 핵심 기능 |
|----|------|------|--------|-----------|
| 1막 | 25% | ~22분 | 18~21 | 주인공 일상 + 촉매 사건 + 첫 전환점 |
| 2막 전반(2a) | 25% | ~22분 | 18~21 | 갈등 에스컬레이션 + 조력자 등장 |
| 2막 후반(2b) | 25% | ~22분 | 18~21 | 미드포인트 반전 + 장르 전환 |
| 3막 | 25% | ~24분 | 16~22 | 클라이맥스 + 해결 + 에필로그 |

target_duration이 90이 아닌 경우, 비례 조정하라.

### 2단계: 원작 사건 재배치 맵

원작의 주요 사건을 나열하고, 각 사건이 어느 막에 배치되는지 매핑하라.

```
원작 사건 → 배치 막 (근거)
```

- 원작 순서와 영화 순서가 다를 수 있음
- 시간 순서 재배치가 필요한 경우 명시
- 삭제/통합/신규 추가가 필요한 사건 명시

### 3단계: 전환점 정의

- **플롯 포인트 I** (1막→2막): 어떤 사건이 주인공을 새로운 세계로 밀어넣는가
- **미드포인트**: 어떤 사건이 가짜 승리/가짜 패배를 만드는가, 장르 전복은 어떻게 일어나는가
- **플롯 포인트 II** (2막→3막): 어떤 사건이 최종 대결로 이끄는가
- **클라이맥스**: 주인공의 최종 능동적 선택은 무엇인가

### 4단계: STEP 0 보완점 반영 확인

STEP 0의 보완점 중 구조 관련 항목을 이 설계에 어떻게 반영했는지 명시하라.

## 출력 형식

```markdown
# STEP 1: 막 구조 및 분량 설계

## 1. 분량 설계

| 막 | 시간 | 씬 수 | 페이지(참고) |
|----|------|--------|-------------|
| 1막 | | | |
| 2막 전반(2a) | | | |
| 2막 후반(2b) | | | |
| 3막 | | | |
| **합계** | **{target_duration}분** | **70~85** | **55~70** |

## 2. 막별 구조

### 1막: [부제]
- 시작점: ...
- 촉매 사건: ...
- 플롯 포인트 I: ...
- 핵심 기능: ...

### 2막 전반(2a): [부제]
- ...

### 2막 후반(2b): [부제]
- 미드포인트: ...
- 장르 전복: ...
- ...

### 3막: [부제]
- 플롯 포인트 II: ...
- 클라이맥스: ...
- 주인공 최종 선택: ...
- 에필로그: ...

## 3. 원작 사건 재배치 맵

| 원작 사건 | 원작 위치 | 영화 배치 | 변경 사유 |
|-----------|-----------|-----------|-----------|
| ... | ... | ... | ... |

## 4. STEP 0 보완점 반영 확인

| 보완점 | 반영 방식 | 반영 위치(막) |
|--------|-----------|--------------|
| ... | ... | ... |
```

## 품질 기준

- [ ] 전체 씬 수가 70~85 범위인가
- [ ] 각 막의 씬 수가 목표 범위 ±3 이내인가
- [ ] 미드포인트가 전체의 47~53% 위치에 있는가
- [ ] 주인공의 능동적 선택 지점이 최소 3곳인가
- [ ] STEP 0 보완점 중 구조 관련 항목이 반영되었는가
- [ ] 원작 사건 재배치 맵이 빠짐없이 작성되었는가

## 주의사항

- 씬 수가 1차 지표이다. 페이지 수는 참고용으로만 제시
- 각 막은 독립적인 미니 구조(시작-중간-끝)를 가져야 한다
- 원작에 없는 사건을 추가할 때는 "추가" 명시, 근거 제시 (보완점 기반)
