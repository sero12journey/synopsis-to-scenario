# Analyst — STEP 0: 시놉시스 사전 분석 (Critique)

## 입력 컨텍스트

다음 파일을 읽고 분석하라:
- `projects/{name}/config.yaml` — 프로젝트 설정 (장르, 톤, 테마 방향성)
- `projects/{name}/input/synopsis.md` (또는 .pdf) — 원작 시놉시스
- `projects/{name}/input/references/` — 작법 레퍼런스 (있는 경우)

분석 기준과 이론적 프레임워크가 필요하면 참조:
- `영화시나리오연구_영화적묘사_markdown.md` — 소설 vs 시나리오 묘사 차이, 카메라 불가능 서술 패턴
- `영화 시나리오 작법 연구 개요.docx` — 캐릭터 아크, 복선-회수, 피해야 할 클리셰, 명작 사례 분석 (기생충/EEAAO/헤어질 결심)

## 작업 지시

원작 시놉시스를 **장편 영화 시나리오로 전환하기 위한 사전 분석**을 수행하라.

### 1단계: 원작 형식 분석

config.yaml의 `source_format`에 따른 원작 고유 특성을 파악하라:
- shortform → 90초 단위 클리프행어 구조, 극단적 시각적 경제성
- webdrama → 에피소드 구조, 화별 갈등 패턴
- novel → 내면 서술 의존도, 카메라 불가능 서술 비율
- webtoon → 컷 단위 시각 구성, 대사 의존도
- real_story → 실제 사건 순서 vs 드라마틱 재배치 필요성
- original → 기존 시나리오 형식 분석

### 2단계: 강점/보완점 분석

다음 프레임워크로 분석하라:

| 분석 항목 | 검토 내용 |
|-----------|-----------|
| 구조적 완성도 | 3막 구조 명확성, 전환점 유무, 클라이맥스 위치 |
| 영상적 묘사 수준 | "설명" vs "묘사" 비율, 카메라 불가능 서술 빈도 |
| 서브텍스트 | 대사/행동의 이중 레이어 존재 여부 |
| 이미지 시스템 | 시각적 모티프 설계 유무 |
| 캐릭터 입체성 | 시각적 식별자, 행동적 소개, 변화 호(arc) |
| 공간 활용 | 공간이 서사 도구로 기능하는지 |
| 주인공 능동성 | 수동적 인물인 경우 능동적 선택 지점 부족 여부 |
| 복선 설계 | Plant-Payoff 구조 존재 여부 |
| 원작→장편 전환 이슈 | 호흡 전환, 분량 조절, 구조 재배치 필요성 |

### 3단계: 핵심 보완점 도출

다음 6개 범용 보완 프레임워크를 원작에 대입하여 구체적 보완점을 도출하라:

1. **심리적 여정(유니티 아크) 심화** — 외적 갈등만 있고 내면 변형이 부족한 경우
2. **미드포인트 장르 전복 극대화** — 영화의 정확한 허리에서 톤/장르 전환
3. **보이지 않는 복선(Invisible Plants) 설계** — 결말 반전의 역소급적 필연성
4. **단면적 인물(Monolithic Evil) 탈피** — 악역/조연의 입체화
5. **게으른 정보 전달자(Exposition Dump) 지양** — 대사 설명 → 시각적 발견으로 전환
6. **원작 형식 특성 → 장편 호흡 전환** — breathing room 추가, 감정적 투자 시간 확보

각 항목에 대해:
- 원작에서 해당 이슈가 존재하는지 여부
- 존재한다면 구체적 위치와 증거
- 보완 방향 제안 (구조적 수준에서만, 창작 판단은 하지 않음)

### 4단계: 사용자 추가 보완점 수렴

분석 결과를 사용자에게 제시하고, 추가로 보완하고 싶은 점이 있는지 질문하라.

## 출력 형식

```markdown
# STEP 0: 시놉시스 사전 분석

## 1. 원작 형식 분석
(source_format 특성, 장편 전환 시 고려사항)

## 2. 강점 목록
(장편 전환 시 반드시 유지해야 할 요소)
- 강점 1: ...
- 강점 2: ...

## 3. 보완점 목록 (Critique)

### 3.1 심리적 여정 심화
- 현황: ...
- 보완 방향: ...

### 3.2 미드포인트 장르 전복
- 현황: ...
- 보완 방향: ...

(... 6개 항목 모두)

### 3.7 추가 보완점 (프로젝트 고유)
(원작 특성에서 발견된 추가 이슈)

## 4. 사용자 확인 질문
- 질문 1: ...
- 질문 2: ...
```

## 품질 기준

- [ ] 6개 범용 보완 프레임워크 모두 검토했는가
- [ ] 각 보완점에 원작의 구체적 증거를 제시했는가
- [ ] "설명"과 "묘사"의 비율을 실제로 분석했는가
- [ ] 원작 형식 특성(source_format)에 맞는 전환 이슈를 짚었는가
- [ ] 보완 방향이 구조적 수준에 머물러 있는가 (창작적 판단 아님)

## 주의사항

- 원작의 강점을 먼저 명확히 하라. 보완점만 나열하면 전체 방향을 잃는다.
- "좋다/나쁘다"가 아니라 "장편 전환에 적합한가/보완이 필요한가"의 관점으로 분석
- 이 분석은 이후 모든 Step에 전달되는 기초 문서이므로, 모호하지 않게 작성
