# Writer — STEP 6: 씬 리스트 (Scene List)

## 입력 컨텍스트

다음 파일을 **모두** 읽어라 (이 Step은 이전 산출물 전체를 참조):
- `projects/{name}/output/step_05_treatment.md` — **트리트먼트 (핵심 입력)**
- `projects/{name}/output/step_00_critique.md` — 보완점 목록
- `projects/{name}/output/step_01_act_structure.md` — 막 구조
- `projects/{name}/output/step_02_beat_sheet.md` — 비트 설계
- `projects/{name}/output/step_03_image_system.md` — 이미지 시스템
- `projects/{name}/output/step_04_characters.md` — 캐릭터

## 작업 지시

**트리트먼트(STEP 5)의 서사를 씬 단위로 분해하라.** 70~85개 씬을 나열한다 (90분 기준). 이것은 시나리오의 "씬별 설계도"이다.

트리트먼트에 서술된 모든 장면이 빠짐없이 씬 리스트에 반영되어야 한다. 트리트먼트의 산문을 읽으며 "여기서 장소가 바뀐다", "여기서 시간이 점프한다", "여기서 새 인물이 등장한다"는 지점마다 씬을 나눈다.

### 각 씬에 포함할 항목

| 항목 | 설명 |
|------|------|
| **씬 번호** | S#1, S#2, ... |
| **장소** | 구체적 공간 (ex: 민준의 원룸 - 거실) |
| **시간** | 낮/밤/새벽/해질녘 등 |
| **등장인물** | 이 씬에 등장하는 인물 |
| **핵심 사건** | 이 씬에서 일어나는 일 (1~2문장) |
| **핵심 이미지** | "이 씬에서 카메라가 반드시 잡아야 할 것" 1개 |
| **비트 번호** | STEP 2의 어느 비트에 해당하는가 |
| **모티프** | STEP 3의 이미지 모티프 해당 여부 (해당 시 모티프명) |
| **복선** | Plant(심기) 또는 Payoff(회수) 해당 여부 |
| **긴장도** | 1~10 |

### 씬 배치 원칙

1. **트리트먼트를 따른다**: STEP 5 트리트먼트의 서사 흐름을 씬 단위로 분해한다. 비트 시트(STEP 2)는 구조적 참조로 활용한다
2. **장소 다양성**: 같은 장소가 3회 이상 연속 반복되지 않게
3. **리듬 변화**: 긴장 씬 후 이완 씬, 대사 씬 후 비주얼 씬
4. **씬 길이 변화**: 짧은 씬(반 페이지)과 긴 씬(2페이지) 섞기
5. **전환 구상**: 씬 간 전환 방식 (컷, 디졸브, 매치컷, 사운드 브릿지 등) 고려
6. **breathing room**: 감정적으로 무거운 씬 후 호흡 공간 확보

### 복선-회수 매핑

씬 리스트 작성 후, 모든 Plant와 Payoff를 매핑하라:

```
Plant (S#XX) → Payoff (S#YY): [복선 내용]
```

모든 Plant에 Payoff가 있어야 하고, 모든 Payoff에 Plant가 있어야 한다.

**보이지 않는 복선(Invisible Plants) 원칙**:
- 복선은 심을 때 관객이 **의식하지 못해야** 한다 (자연스러운 소품/대사/행동으로 위장)
- 회수 시 관객이 "아, 그때 그것이!" 하며 역소급적 필연성을 느껴야 한다
- Deus ex Machina(데우스 엑스 마키나) 회피: 모든 결말의 결정적 요소는 반드시 사전에 Plant 되어야 함
- 사례: 《겟 아웃》의 찻잔, 《백 투 더 퓨처》의 시계탑 전단지

```markdown
# STEP 6: 씬 리스트

### S#2
...

(70~85개 씬 모두)

## 씬 전환 구상

| S# → S# | 전환 방식 | 비고 |
|----------|-----------|------|
| S#1 → S#2 | 컷 | ... |
| ... | 매치컷 | ... |

## 복선-회수 매핑

| Plant | Payoff | 내용 |
|-------|--------|------|
| S#XX | S#YY | ... |
| S#XX | S#YY | ... |

## 분량 점검

| 막 | 씬 수 | 목표 | 차이 |
|----|--------|------|------|
| 1막 | | 18~21 | |
| 2막 전반 | | 18~21 | |
| 2막 후반 | | 18~21 | |
| 3막 | | 16~22 | |
| **합계** | | **70~85** | |
```

## 품질 기준

- [ ] 총 씬 수가 70~85 범위인가
- [ ] 각 막의 씬 수가 STEP 1의 설계와 일치하는가
- [ ] 모든 씬에 "핵심 이미지"가 있는가
- [ ] 모든 Plant에 대응 Payoff가 있는가, 그 역도 확인
- [ ] 이미지 모티프가 3개 막에 고르게 분포하는가
- [ ] breathing room 씬이 최소 3곳 있는가
- [ ] 같은 장소 3회 이상 연속 반복이 없는가

## 주의사항

- "핵심 이미지"는 가장 중요한 항목이다. 이것이 있으면 씬의 존재 이유가 명확해진다.
- 씬 하나에 사건을 2개 이상 넣지 말 것. 1씬 = 1사건이 원칙.
- 이 씬 리스트는 STEP 5(트리트먼트)를 기반으로 작성되며, STEP 7(1고)의 직접적 설계도가 된다.
- 트리트먼트에 서술된 장면이 씬 리스트에 누락되면 안 된다. 트리트먼트와 씬 리스트 사이에 불일치가 없어야 한다.
